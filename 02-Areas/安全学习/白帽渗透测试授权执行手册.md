# 白帽渗透测试授权执行手册

## 完整流程图

```text
准备与授权 → 被动信息搜集 → 主动资产扫描 → 服务枚举
→ Web/API 枚举 → 漏洞假设 → 低风险验证 → 初始访问评估
→ 提权风险评估 → 影响分析 → 修复建议 → 复测 → 报告归档
```

## 各阶段要点

### 阶段零：授权
确认 IP/域名范围、时间窗口、允许工具、禁止动作、应急联系人。
实验目录：`mkdir -p ~/pentest-lab/{scans,web,evidence,notes,reports}`

### 阶段一：被动信息搜集
- 子域名：`subfinder -d example.local`
- 证书透明度 / GitHub 搜索 / 官网文档

### 阶段二：主动资产扫描
```bash
nmap -sn 192.168.56.0/24 -oN scans/01-host-discovery.txt
nmap -sV 192.168.56.20 -oN scans/02-service-basic.txt
```

### 阶段四：Web/API 枚举
记录：首页、登录、搜索、上传、管理后台、API、错误页面、Input 参数点

### 阶段五：漏洞验证流程
```text
发现输入点 → 判断数据流向 → 提出漏洞假设
→ 设计低风险测试 → 观察响应差异 → 记录证据 → 修复建议
```

### 阶段六-七：初始访问 + 提权评估
Linux 枚举（仅靶场）: `sudo -l`, `find / -perm -4000`, `getcap -r /`, `crontab -l`

### 阶段八：扫描器使用
- **Nuclei**: 基于模板检查，必须人工验证
- **ZAP**: 先被动再主动，范围确认后再扫描
- **Metasploit**: 优先 `check` 模式，不跑破坏性 payload

## 报告模板

```text
项目名称 / 测试时间 / 授权范围
一、执行摘要
二、资产清单
三、测试方法
四、发现详情（编号/标题/等级/目标/证据/影响/修复）
五、修复优先级（P0-P3）
六、防守建议（代码/配置/权限/监控）
七、附录
```

## 红蓝对照

| 红队阶段 | 蓝队防守 |
|---------|---------|
| 信息搜集 | 资产台账、Git 泄露监控 |
| 端口扫描 | 暴露面收敛、防火墙 |
| 服务枚举 | 版本管理、banner 收敛 |
| Web 漏洞 | 安全编码、WAF、日志告警 |
| 提权 | sudoers/SUID 审计、补丁 |
| 横向移动 | 网络分段、零信任 |
