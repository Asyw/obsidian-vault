# 主流攻击方式学习路线

## 攻击链

```text
授权确认 → 信息搜集 → 资产识别 → 端口扫描
→ Web/API 枚举 → 漏洞假设 → 非破坏性验证
→ 初始访问评估 → 提权评估 → 横向移动评估
→ 修复建议 → 复测 → 报告
```

## 攻击面分类

### Web 应用攻击
SQLi / XSS / CSRF / 文件上传 / 命令注入 / SSRF / 反序列化 / 模板注入 / IDOR

**推荐靶场**: PortSwigger Academy / Juice Shop / DVWA / WebGoat

### API 攻击
BOLA/IDOR / Broken Auth / Mass Assignment / SSRF / GraphQL 滥用

**推荐**: OWASP crAPI / PortSwigger API labs

### 服务与主机
SSH / RDP / SMB / FTP / 数据库暴露 / 管理面（Jenkins/Grafana/Kibana）

### 云与容器
云存储公开 / IAM 权限过大 / Docker socket / K8s API / CI/CD 泄露

## 信息搜集

**被动**: 域名、子域名、IP 段、GitHub 泄露、技术栈指纹、证书记录
**主动**（仅靶场）: nmap 主机发现 + 服务识别

## Web 漏洞学习顺序

1. HTTP 基础（curl、状态码、Header）
2. SQL 注入根因（拼接 vs 参数化）
3. 文件上传防守配置
4. 命令注入
5. SSRF
6. 访问控制缺陷（越权）

## 提权思路

**Linux 常见提权类别**: sudo 配置错误、SUID、可写路径、PATH 劫持、cron、capabilities、Docker 组、NFS、内核漏洞

**枚举命令**: `sudo -l`, `find / -perm -4000`, `getcap -r /`, `crontab -l`, `ss -tulpen`

## 推荐资料

- OWASP Top 10 / WSTG / Cheat Sheet Series
- PortSwigger Web Security Academy
- MITRE ATT&CK
- Hacking: The Art of Exploitation（书籍）
- HackTricks / GTFOBins

## 三个月计划

- **1-2 周**: Linux + 网络
- **3-4 周**: 扫描与枚举（Metasploitable / VulnHub）
- **5-8 周**: Web 漏洞（PortSwigger / Juice Shop / DVWA）
- **9-10 周**: 提权思路（HTB Academy / TryHackMe）
- **11-12 周**: 综合报告 + 完整靶场演练

> 推荐学习顺序：Linux → 网络/HTTP → Nmap → Burp/ZAP → SQLi → XSS → 越权 → 文件上传 → SSRF → 提权枚举 → 报告 → 修复复测 → 防守监控
