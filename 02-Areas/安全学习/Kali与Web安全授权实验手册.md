# Kali 与 Web 安全授权实验手册

## 核心原则

安全测试的第一条能力，是知道边界在哪里。

- **可以做**: 自有靶场、授权测试、CTF、自己的代码
- **不可以做**: 扫描公网 IP、跑 sqlmap 打真实网站、拿 shell、窃取数据

## Kali 的正确用法

Kali 是安全测试工具集合，不是"黑客系统"。正确方法：

```text
1. 明确授权范围
2. 识别资产和暴露面
3. 枚举服务和版本
4. 建立漏洞假设
5. 做低风险验证
6. 评估影响
7. 给出修复建议
8. 复测确认修复
```

## 攻击面地图

- **暴露服务**: SSH、RDP、数据库、管理后台、VPN
- **Web 应用**: SQL 注入、XSS、SSRF、文件上传、反序列化、命令注入
- **已知漏洞**: 公开 CVE、旧组件、未打补丁
- **配置错误**: 目录暴露、调试模式、过宽权限

## 标准流程

```text
授权确认 → 资产识别 → 端口扫描 → 服务枚举
→ Web 枚举 → 漏洞假设 → 低风险验证
→ 影响评估 → 修复建议 → 复测 → 报告
```

## SQL 注入

根因：用户输入被拼接到 SQL 结构。修复：参数化查询。

## Web Shell 风险链

```text
上传点可控 → 文件类型校验失败 → 保存到 Web 可访问目录
→ 服务器把文件当脚本执行 → 攻击者获得应用进程权限
```

## 常见漏洞速记

| 类型 | 根因 | 修复 |
|------|------|------|
| SQL 注入 | 输入进入 SQL 结构 | 参数化查询 |
| 命令注入 | 输入进入系统命令 | 避免 shell 调用 |
| XSS | 输入作为 HTML/JS 输出 | 输出编码 + CSP |
| SSRF | 服务器请求用户可控 URL | URL 白名单 |
| 文件上传 | 文件类型/路径控制不足 | 白名单 + 根目录外存储 |
| 越权 | 服务端未鉴权 | 统一权限检查 |

## 四周路线

- **第 1 周**: Linux + 网络基础
- **第 2 周**: 扫描与枚举
- **第 3 周**: Web 漏洞根因
- **第 4 周**: 报告与加固
