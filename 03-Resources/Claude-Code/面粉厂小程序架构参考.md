# 面粉厂小程序架构参考

## 项目结构

```
flour-mill-miniapp/
├── app.js              # App 入口
├── app.json            # 4 tabBar + 6 非 tab 页面
├── app.wxss            # 全局样式
├── utils/
│   ├── api.js          # ★ 核心: JWT 封装 + mock 数据全集
│   ├── products.js     # 本地 fallback 商品数据
│   ├── localOrders.js  # 本地 fallback 订单 CRUD
│   ├── orderRecords.js # 隐藏已取消订单
│   ├── payment.js      # wx.requestPayment 封装
│   └── lang.js         # 中/维双语字典
├── images/
│   └── products/       # 5kg.jpg / 10kg.jpg / 25kg.jpg
└── pages/
    ├── index/          # 首页 — 分类+搜索+购物车
    ├── order/          # 下单页
    ├── orderlist/      # 订单列表
    ├── orderdetail/    # 订单详情
    ├── profile/        # 我的
    ├── address/        # 地址编辑
    ├── admin/          # 管理后台 (6标签)
    ├── coupons/        # 优惠券
    ├── invite/         # 邀请码
    └── prepaid/        # 储值卡
```

## 数据流

```
USE_MOCK = false
  └─ api.js → wx.request → https://xinjiangyuwen.cn/api/xxx → JWT Bearer
USE_MOCK = true
  └─ api.js → mockRequest() → 返回 mock 数据

登录:
  app.login() → wx.login() → POST /api/login {code} → JWT → 存 Storage
  401 → 自动重登录 + 重试 (authRetried 防死循环)

购物车:
  app.globalData.cart → index.js 和 order.js 共享操作
```

## 业务规则

- 配送时间: 1-9袋→一周内, 10-19→3天内, 20+→1天内
- 配送费: 固定 0 元
- 订单状态: pending→confirmed→delivered→completed (或→cancelled)
- 支付方式: wechat / credit
- 积分: ¥1=1积分, 100积分=¥1, 单笔最多抵扣50%

## 约束

1. 所有页面通过 `utils/api.js` 调用后端
2. api 返回格式: `{code: 0, data: {...}}`
3. 地址格式: `"张三\n138xxx\n详细地址"`
4. 不要加配送费逻辑
5. 修改后端需要同步部署到服务器
