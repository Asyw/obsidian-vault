# 知识库同步指南

## 概述

本知识库（Obsidian AI 记忆知识库）在 **Mac mini** 和 **Windows** 之间通过 Git 进行双向同步。

GitHub 仓库：`https://github.com/Asyw/obsidian-vault.git`（分支：`master`）

## 各机器路径

| 机器 | 用户 | 知识库路径 | Codex 用户 |
|------|------|-----------|-----------|
| Mac mini（局域网） | `xiaosong` | `/Users/xiaosong/Documents/Obsidian Vault` | 同账户 |
| Windows | `NINGMEI` | `C:\Users\NINGMEI\Documents\Obsidian` | 同账户 |

Windows 上 `C:\Users\NINGMEI\Documents\Obsidian` 实际通过 NTFS junction 指向 `D:\C盘迁移数据\NINGMEI\用户文件\Documents\Obsidian`。

## 同步方式

使用 **Git** + **GitHub** 进行同步。

### 首次设置

```powershell
# Windows
cd C:\Users\NINGMEI\Documents\Obsidian
git clone https://github.com/Asyw/obsidian-vault.git .
```

Mac mini 同理，clone 到 `/Users/xiaosong/Documents/Obsidian Vault`。

### 日常同步流程

在任意一台机器上编辑笔记后：

```powershell
cd <知识库目录>
git pull origin master          # 先拉取远程改动
git add -A                      # 暂存本地改动
git commit -m "描述你的改动"      # 提交
git push origin master          # 推送到 GitHub
```

在另一台机器上：

```powershell
cd <知识库目录>
git pull origin master          # 拉取最新改动
```

### Codex / AI Agent 模式下同步

当我在 Codex 中需要同步知识库时，执行以下步骤：

1. 读取本文件获取同步参数
2. `git pull origin master`（获取对方机器的改动）
3. 如果需要，`git add -A && git commit && git push origin master`
4. 记录同步操作到 projectmem

## 注意事项

- **不要提交含密钥/Token 的文件**：GitHub 启用了 Push Protection，会在推送前拦截。`.env`、`config.json` 等含 API 密钥的文件需要检查。
- **Obsidian 可能会占用 Git 索引锁**：如果在 Obsidian 打开时 `git add` 失败，先关闭 Obsidian 再操作。
- **提交前检查是否含敏感信息**：`git diff --staged` 预览改动。
- **双向同步前务必先 pull**：避免推送冲突。

## 故障排查

| 问题 | 解决方案 |
|------|----------|
| GitHub 推送被拒（Push Protection） | 检查提交文件是否含 API Key，`git rm --cached` 移除后 `git commit --amend` |
| `git add` 失败（index.lock） | 关闭 Obsidian，删除 `.git/index.lock` |
| `git pull` 冲突 | 手动解决冲突后 `git add` + `git commit` |
| 连接到 Mac mini 失败 | 确认 Mac mini 在线（ping 192.168.1.223），SSH key 有效 |
